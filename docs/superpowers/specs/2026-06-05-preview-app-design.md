# TDGame Preview App — Design Spec
**Date:** 2026-06-05  
**Status:** Approved

---

## 1. Overview

A private preview portal that lets clients of TDGame Studio review their Art, Animation, and VFX deliverables. Internal team uploads and manages assets; clients log in to view, comment, and download. Projects can optionally be shared via a public link without login.

---

## 2. Users & Roles

| Role | Access |
|---|---|
| `internal` | Full dashboard: manage clients, projects, upload assets, toggle share links |
| `client` | Read-only portal: view their own projects, comment, download files |
| `anonymous` | Read-only share page via token URL (no login required, must be enabled per project) |

---

## 3. Architecture

```
VPS Ubuntu
├── nginx (reverse proxy, SSL)
└── Docker: Next.js 14 App Router
      ├── /app/(auth)/login
      ├── /app/(dashboard)/*   → Internal team
      ├── /app/(portal)/*      → Client view
      └── /app/share/[token]   → Public share

External services:
├── Supabase — Auth (email/password), PostgreSQL, Realtime (comments)
└── Cloudflare R2 — File storage (art, spine, vfx assets)
```

**Tech stack:**
- Frontend: Next.js 14 (App Router), Tailwind CSS, shadcn/ui
- Backend: Supabase (Auth + PostgreSQL + RLS)
- Storage: Cloudflare R2 (presigned URLs, uploads direct from browser)
- Deploy: Docker + nginx on existing VPS Ubuntu

## 4. Database Schema (Supabase, prefix `Prv_`)

```sql
-- Extends auth.users
Prv_profiles (
  id           uuid PK → auth.users.id,
  role         enum('internal', 'client'),
  client_id    uuid FK → Prv_clients (null if internal),
  display_name text,
  created_at   timestamptz
)

-- Client companies
Prv_clients (
  id         uuid PK,
  name       text,          -- "GameStudio X"
  slug       text UNIQUE,   -- "gamestudio-x"
  logo_url   text,
  created_at timestamptz
)

-- Projects
Prv_projects (
  id            uuid PK,
  client_id     uuid FK → Prv_clients,
  name          text,
  description   text,
  status        enum('active', 'archived') DEFAULT 'active',
  spine_version text,       -- "4.2" | "3.8" | null (project-level, all Animation assets use this)
  share_enabled bool DEFAULT false,
  share_token   text UNIQUE, -- random UUID, generated when share is enabled
  created_at    timestamptz
)

-- Assets
Prv_assets (
  id           uuid PK,
  project_id   uuid FK → Prv_projects,
  service_type enum('art', 'animation', 'vfx'),
  name         text,
  r2_key       text,        -- full path in R2 bucket
  file_type    text,        -- 'png'|'jpg'|'psd'|'spine'|'gif'|'mp4'|'unitypackage'
  metadata     jsonb,       -- see below
  sort_order   int DEFAULT 0,
  created_at   timestamptz
)

-- Comments (project-level or asset-level)
Prv_comments (
  id         uuid PK,
  project_id uuid FK → Prv_projects,
  asset_id   uuid FK → Prv_assets (null = project-level comment),
  author_id  uuid FK → auth.users,
  content    text,
  created_at timestamptz
)
```

**`metadata` jsonb examples:**
```json
// Spine animation asset
{ "animations": ["walk", "idle", "attack", "death"], "skins": ["default", "red"] }

// VFX video
{ "duration_sec": 12, "fps": 60, "resolution": "1920x1080" }

// Art PSD
{ "thumbnail_key": "assets/xxx/thumbnail.png", "width": 4096, "height": 4096 }
```

**Row Level Security:**
- `client` role: SELECT only on rows where `client_id` matches their profile
- `internal` role: full CRUD
- Share token: Supabase RLS policy allows anonymous SELECT when `share_token` matches and `share_enabled = true`

## 5. Asset Preview System

### Art (PNG / JPG / PSD)
- PNG/JPG: Rendered directly via R2 presigned URL in `<img>` tag
- PSD: Display pre-exported thumbnail PNG; Download button for original PSD
- Click any image → fullscreen lightbox

### Animation — Spine Web Player
- Spine version is **per-project** (stored in `Prv_projects.spine_version`)
- Upload structure in R2:
  ```
  /assets/{asset_id}/
    skeleton.json (or .skel)
    skeleton.atlas
    textures/skeleton.png
  ```
- Runtime JS loaded dynamically by version: `spine-player-{version}.js`
- Viewer features: play/pause/loop, animation selector (from `metadata.animations[]`), skin selector, background toggle (checkerboard / solid color)

### VFX
| File type | Preview method |
|---|---|
| GIF | `<img>` autoplay |
| MP4 / WebM | `<video>` with controls, loop |
| Unity Package | Icon + filename + Download button only |

### Download / Export
All downloads via **R2 signed URL** (1-hour expiry), generated by Next.js API route — R2 credentials never exposed to client.

| Asset | Download |
|---|---|
| Art PNG/JPG | Original file |
| Art PSD | Original PSD |
| Spine | ZIP of all files (json + atlas + textures) |
| VFX GIF/Video | Original file |
| Unity Package | .unitypackage file |

## 6. Upload Flow (Internal Team)

```
1. Select Client → Project → Service type
2. Drag & drop file(s)
3. Next.js API route generates R2 presigned PUT URL
4. Browser uploads directly to R2 (bypasses server for large files)
5. After upload: call API to save asset record in Supabase
6. For Spine: server parses .atlas file to extract animations[] and skins[]
7. For PSD: server generates thumbnail PNG via sharp (or team provides manually)
```

## 7. UI Structure

### Internal Dashboard (`/dashboard`)
```
Sidebar: Client list
/dashboard                      → Overview: stats, recent uploads
/dashboard/clients              → Client list
/dashboard/clients/[id]         → Client detail + project list
/dashboard/clients/[id]/[pid]   → Project detail:
    Tab: Art        → grid, upload, reorder, delete
    Tab: Animation  → grid, upload, delete
    Tab: VFX        → grid, upload, delete
    Tab: Settings   → spine_version, share link toggle, project status
```

### Client Portal (`/portal`)
```
/portal                         → Grid of client's projects
/portal/[project_slug]          → Project detail:
    Tab: Art        → image grid → lightbox on click
    Tab: Animation  → thumbnail grid → Spine viewer modal on click
    Tab: VFX        → preview grid → video/gif modal on click
    Tab: Comments   → project-level thread; asset threads accessible from asset view
```

### Public Share (`/share/[token]`)
- Same layout as Client Portal
- No login required; returns 404 if `share_enabled = false`
- Download buttons: enabled by default (not configurable in v1)
- Minimal header with studio branding

---

## 8. Comments System
- Project-level: visible in project Comments tab
- Asset-level: visible inside the asset viewer modal (slide-in panel)
- Realtime updates via Supabase Realtime subscriptions
- Authors identified by `display_name` from `Prv_profiles`
- Internal team comments visible to clients (no internal-only threads in v1)

---

## 9. R2 Bucket Structure
```
Bucket: tdgames-preview
/assets/{asset_id}/           → spine bundles, psd+thumbnail
/assets/{asset_id}.{ext}      → single-file assets (png, jpg, gif, mp4, unitypackage)
/clients/{client_id}/logo.{ext}
```

## 10. Deployment

```yaml
# docker-compose.yml
services:
  app:
    build: .
    env_file: .env.production
    ports: ["3000:3000"]
    restart: unless-stopped

# nginx: proxy pass to :3000, SSL via certbot
```

**Environment variables:**
```
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
R2_ACCOUNT_ID
R2_ACCESS_KEY_ID
R2_SECRET_ACCESS_KEY
R2_BUCKET_NAME
R2_PUBLIC_URL   # custom domain or r2.dev URL
```
